# XXX-001: Namespaced Installation

## Context

Argo Workflows supports three installation modes:

- **Cluster install** — controller watches and executes workflows across all namespaces. Uses ClusterRole/ClusterRoleBinding. Default in official release manifests.
- **Namespace install** — controller only executes workflows in the namespace it is installed in (typically `argo`). Uses Role/RoleBinding scoped to that namespace.
- **Managed namespace install** — controller is installed in one namespace but executes workflows in a separate designated namespace.

## Decision

Use a **namespaced install**, scoped to the `sys-argo-workflows` namespace.

This means:

- The controller Role and server Role are namespace-scoped, not cluster-wide
- Workflows are submitted to and executed within the `sys-argo-workflows` namespace

## Consequences

- Blast radius of controller and server permissions is limited to the `sys-argo-workflows` namespace
- The controller cannot inadvertently interact with resources in other namespaces
