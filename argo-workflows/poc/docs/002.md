# XXX-002: CRD Installation Method

## Context

Argo Workflows CRDs (clusterworkflowtemplates, cronworkflows, workflows, workflowtemplates) are quite large, when installed via `kubectl apply`, the full manifest is stored in the `kubectl.kubernetes.io/last-applied-configuration` annotation, which exceeds the Kubernetes 256KB annotation size limit, resulting in:

```
Error from server (Invalid): error when creating "cluster": CustomResourceDefinition.apiextensions.k8s.io
"workflows.argoproj.io" is invalid: metadata.annotations: Too long: may not be more than 262144 bytes
```

## Decision

Install CRDs using server-side apply (`--server-side` flag) rather than standard client-side `kubectl apply`.

```bash
kubectl apply --server-side -k cluster
```

## Consequences

- CRD installation and upgrades must always use `--server-side` unless we patch it to reduce annotation
- Server-side apply moves tracking to the server, avoiding the annotation size limit entirely
