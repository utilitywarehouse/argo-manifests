# XXX-004: Controller, Server RBAC and PriorityClass

## Context

The default Argo Workflows RBAC roles are intentionally permissive to cover all possible use cases out of the box. For our namespaced install the following default permissions were identified as unnecessary and were raised as concerns:

- `pods/exec` on both controller and server — allows shelling into any pod in the namespace
- `secrets/get` on controller — allows reading any secret in the namespace
- `persistentvolumeclaims` create/update/delete on controller — allows creating and destroying any PVC
- `configmaps` create/update on server — server has no need to write configmaps
- `secrets/create` on server — server has no need to create secrets

## Decision

Could be left as is because it is scoped to the namespace. [see xxx-001](./001.md). The following patches have been created to allow for us to iteratively review and adjust.

Regarding PriorityClass, again as it is a namespace install, I think we are ok, if not then we can ammend appropriately

- [controller](../../namespaced/controller/patches/role.yaml)
- [server](../../namespaced/server/patches/role.yaml)
