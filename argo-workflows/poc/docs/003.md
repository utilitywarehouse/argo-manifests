# XXX-003: Argo Server Authentication Mode

## Context

Argo Server supports multiple authentication modes:

- `client` — users authenticate with their own kubeconfig token, passed via the UI login screen
- `server` — the server uses its own service account, no user token required
- `sso` — delegates to an external SSO provider

The default mode requires users to supply a Bearer token in the UI, this is unnecessary for POC but I imagine we will want some kind of authentication in dev. I believe we can use our set up with ArgoCD for SSO though I wont be able to confirm locally, we can test this once we get into `exp-1-merit`.

- https://argo-workflows.readthedocs.io/en/latest/argo-server-sso-argocd/

## Decision

Run Argo Server in `--auth-mode=server` for now. Look into SSO later

```yaml
args:
  - server
  - --namespaced
  - --auth-mode=server
```
